# Sprint 2 — Crew Workflow Clarity

## Goal
Make the crew workflow understandable and guided, not just functional.

## Scope
- Care timeline/progression
- Next-step guidance
- Better grouping of encounter/observation/intervention/handover actions
- Form usability improvements

## Tasks
- Add care timeline to crew incident detail
- Add "next step" guidance panel
- Group forms by workflow stage
- Add sensible defaults where appropriate (e.g. current time)
- Improve layout readability and progression clarity

## Acceptance Criteria
- Crew UI clearly shows current workflow stage
- Crew can tell what action is needed next
- Forms are grouped and easier to use
- Existing write flows still work
- Frontend tests cover major render/path conditions

## Non-goals
- No offline mode
- No mobile-native app implementation
- No backend workflow rule changes
