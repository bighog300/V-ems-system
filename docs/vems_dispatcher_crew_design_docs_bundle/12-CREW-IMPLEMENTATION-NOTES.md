# Crew Mobile Implementation Notes

## Suggested implementation phases

### Phase A
- Create mobile-first assignment view
- Add sticky primary action bar
- Simplify current assignment card

### Phase B
- Split patient and handover flows into clearer sections
- Reduce typing with structured controls
- Improve narrow-screen layout and spacing

### Phase C
- Add degraded-connectivity messaging and UX
- Improve focus, touch target sizing, and accessibility

## Acceptance criteria
- Crew can progress through a call on a phone without desktop zooming
- Next action is always obvious
- Handover can be completed in a small number of taps
- High-priority information is visible above the fold
