# Dispatcher Interaction Flows

## Flow 1 — Assign a crew
1. Dispatcher sees unassigned incident in Critical or Pending Assignment
2. Selects incident card
3. Side panel shows eligible/available units
4. Dispatcher assigns a unit in one primary action
5. Card moves group and briefly highlights change
6. Confirmation message appears inline, not as a blocking modal

## Flow 2 — Escalate blocked incident
1. Dispatcher sees blocked/overdue card
2. Card already exposes escalation reason
3. Dispatcher opens side panel
4. Adds escalation note or flags supervisor review
5. Board updates escalation badge and activity timeline
