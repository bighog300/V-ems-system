# 05 - Screen-by-Screen UX Specification

## UX design principles
1. Critical actions must be obvious and fast.
2. High-priority incidents must always be visually distinguishable.
3. Users should not need to navigate through multiple systems manually.
4. Every operational screen should expose the current Incident ID.
5. Clinical data entry should be structured, concise, and field-friendly.
6. Mobile screens must support field use with intermittent connectivity.

## Critical screen notes
### Call Handler Console
- keyboard-friendly
- minimal mandatory fields before incident creation
- duplicate incident suggestions shown without blocking workflow

### Dispatcher Board
- waiting incidents by priority
- available vehicles and crews
- assign/reassign/stand-down actions
- near real-time visibility

### Crew Mobile
- large touch targets
- visible offline/sync state
- quick status buttons
- patient and clinical actions from incident screen

### Handover / Disposition
- required fields change based on disposition
- handover completion should be explicit and confirmed
