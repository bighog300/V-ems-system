# Phase C3 — Logistics Dashboard Scaffold

## Goal
Create the first logistics/stock-focused operational surface.

## Initial dashboard sections
- stock-linked intervention summary
- stock sync health
- candidate replenishment signals (scaffold only)

## Scope constraints
- visibility first
- no procurement/supply-chain logic
- reuse existing stock sync visibility work

## UX requirements
- readable cards/tables
- recent failure visibility
- drill-down links where possible

## Testing
- render tests
- empty/error-state handling
- no live Vtiger dependency

## Definition of done
- logistics dashboard scaffold exists
- stock-related outcomes are visible
- tests pass
