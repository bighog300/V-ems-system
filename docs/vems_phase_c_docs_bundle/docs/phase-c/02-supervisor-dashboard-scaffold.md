# Phase C2 — Supervisor Dashboard Scaffold

## Goal
Create the first supervisor-focused operational surface.

## Initial dashboard sections
- delayed / aging incidents
- closure-blocked incidents
- sync failure summary
- workflow health indicators

## Scope constraints
- scaffold first
- read-only summary surface
- use existing endpoints where possible
- do not invent data

## UX requirements
- concise cards/sections
- obvious counts and exception lists
- click-through to incident detail where possible

## Testing
- render tests
- transformation tests
- empty/error-state tests

## Definition of done
- supervisor dashboard scaffold exists
- shows useful oversight data
- tests pass
